# Documentación de Implementaciones

Esta carpeta contenida en `.docs/` contiene archivos Markdown con los detalles técnicos, decisiones de arquitectura y planes de implementación realizados durante el desarrollo del proyecto PathWord.

## Reglas Fundamentales de Desarrollo
> [!IMPORTANT]
> - **Autorización Obligatoria:** No se permite realizar cambios en el código sin la autorización previa y explícita del usuario.
> - **Clean Architecture:** Se debe respetar rigurosamente la arquitectura de capas ya establecida en el proyecto (Domain, Data, Presentation).
> - **Clean Code:** El código debe seguir los principios de Clean Code para garantizar su legibilidad, mantenibilidad y calidad.
